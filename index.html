JavaScript is a High-Level, Object-Oriented, Single-threaded programming language. As it's single-threaded, it can only execute one command at a time and in a specific order (from top to bottom). JavaScript runs in the browser with the help of an engine like V8, which compiles the JavaScript code one by one through the JIT(Just-In-Time) compiler. When a function is invoked in JavaScript, JavaScript creates an execution context. It stores the function in the call stack. If another function is called inside the function, it will be placed above the call stack and executed first. Finally, the main function will execute. Simply says, it applies the last in first out method.

HTML web storage provides two objects for storing data on the client: localStorage() and sessionStorage(). The main features of localStorage are:
  • There is no expiration date for the data stored in localStorage. Data remains after the browser restarts, even after an OS reboot.
  • Each tab and window with the same origin has access to local storage. So if we set data in one window, the change becomes visible in another one.
  • The storage capacity of local storage is 5MB/10MB(depending on the browser)
The main features of localStorage are:
  • There is only one sessionStorage per tab in the browser. When we close the tab or browser, sessionStorage will be gone.
  • Another tab with the same page will have a different storage.
  • The storage capacity of session storage is 5MB(depending on the browser)
